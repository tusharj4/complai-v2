## BACKEND

### Core Pipeline
- [x] Phase 0: FastAPI skeleton, auth, DB models, Celery, Docker Compose

- [x] Phase 2: Kafka streaming, manual override, retry endpoint, monitoring, performance tests

- [x] Phase 4: ROC scraper complete, bank statement multi-format, webhooks, rate limiting, Redis caching

### Test Coverage
- [x] 44+ unit tests passing (`pytest tests/ -v`)
- [x] Performance benchmarks passing (single classify <1s, 100 batch <10s)
- [x] Load test written (Locust — `tests/load_test.py`)
- [ ] Load test run with 50 concurrent users in staging environment
- [ ] P99 response time < 800ms under load
- [ ] Error rate < 0.1% under load

### Security
- [ ] JWT_SECRET changed from default value (min 32 chars, cryptographically random)
- [ ] CORS `allow_origins` restricted to portal domain (not `"*"`) in production
- [ ] All secrets in AWS Secrets Manager or SSM (not in `.env` committed to git)
- [x] Partner isolation enforced on all routes (partner_id filter on every query)
- [x] Authorization checks on all company/document endpoints (403 on cross-partner access)
- [ ] Rate limiting tested — 429 returned after limit exceeded
- [x] HMAC-SHA256 webhook signing implemented

### Reliability
- [x] DB connection pooling configured (pool_size=10, max_overflow=20, pool_pre_ping=True)
- [x] Celery DLQ configured (failed tasks routed to `dlq` queue)
- [x] Kafka graceful fallback (events logged locally when Kafka unavailable)
- [x] Scraper retry with exponential backoff (2 retries, 30s/60s delays)
- [x] Webhook retry (3 attempts, 5s/30s/120s delays)
- [x] Health check endpoint (`/health`) testing DB + Redis + Celery
- [x] Backup scripts (`scripts/backup.sh`, `scripts/restore.sh`)
- [ ] Backup tested — verified restoration from backup file
- [ ] Alembic migrations tested on fresh PostgreSQL instance

### Monitoring
- [ ] Datadog account configured with API keys in prod env
- [ ] Datadog alerts created:
  - [ ] Scraper success rate < 90% → Medium
  - [ ] Classification latency > 5s → Medium
  - [ ] Error rate > 5% → High
  - [ ] Queue depth > 1000 → High
  - [ ] Uptime < 99% → Critical
- [x] Slack webhook configured for critical alerts
- [x] Alert runbooks written (`docs/TROUBLESHOOTING.md`)
